# Content Update - 記事更新・改善・新規作成

## 概要
既存ブログ記事の品質チェック・改善と、新規記事の自動生成を行う。変更後はビルド確認してgit pushまで実行。

## プロジェクトパス
- ブログ記事: `content/blog/*.md`
- トピックリスト: `content/blog-topics.json`
- ブログ画像: `public/images/blog/`

## 実行手順

### モード1: 既存記事の改善

#### 1. 全記事をスキャン
```bash
ls content/blog/*.md
```

#### 2. 各記事をチェック（品質基準）
Read各記事のfrontmatterと本文を確認:

| チェック項目 | 基準 | 対応 |
|---|---|---|
| H2見出しの数 | 4つ以上 | 不足なら追加 |
| 内部リンク /course/ | 3本以上 | 不足なら追加（楽天コースID使用） |
| 内部リンク /shindan | 2本以上 | 不足なら追加 |
| 文字数 | 2000〜3000字 | 薄い場合は内容追加 |
| 日付 | 1ヶ月以上前なら更新 | dateフィールドを今日に更新 |
| 画像 | image フィールドあり | なければ追加 |
| マークダウンテーブル | 1つ以上 | なければ追加 |

#### 3. 改善を適用
Editツールで各記事を修正。主な改善:
- 内部リンク追加（コースID: 千葉120XXX、埼玉110XXX、茨城80XXX、栃木90XXX、群馬100XXX、東京130XXX、神奈川140XXX）
- 薄いセクションの補強
- 新しいH2/H3の追加
- 日付の更新

### モード2: 新規記事の生成

#### 1. 未作成トピック特定
```javascript
// blog-topics.json から content/blog/ に存在しないslugを抽出
```

#### 2. 記事を生成
frontmatter形式:
```markdown
---
title: "{タイトル}"
description: "{120〜160文字のSEO説明文}"
date: "{今日の日付 YYYY-MM-DD}"
category: "{カテゴリ}"
tags: "{カンマ区切りタグ}"
image: "/images/blog/{slug}.jpg"
---
```

本文の要件:
- H2見出し: 4つ以上
- 内部リンク /course/XXXXXX: 3本以上（実在するコースID）
- 内部リンク /shindan: 2本以上
- マークダウンテーブル: 1つ以上
- 文字数: 2000〜3000字
- ターゲット: 30代男性ゴルファー
- 口調: ですます調、実用的

#### 3. アイキャッチ画像
既存画像からコピー:
```bash
cp public/images/blog/chiba-golf-cheap.jpg public/images/blog/{new-slug}.jpg
```

### 共通: ビルド＆プッシュ

```bash
npm run build  # エラーがないことを確認
git add -A
git commit -m "コンテンツ更新: {変更内容の要約}"
git push
```

ビルドエラーが出たら修正してから再度push。
